## 2025-03-15 - [Added Input Length Limits to Contact Form]
**Vulnerability:** Missing input length limits on the contact form (`ContactForm.tsx`). Fields like name, email, company, and message lacked `maxLength` constraints and equivalent limits in the validation logic, which creates a Denial of Service (DoS) risk (Medium Priority) where excessively large payloads could be submitted to the external form handling API (Formspree).
**Learning:** External API integrations for forms often omit basic HTML and client-side payload limits because they depend on the API endpoint to handle abusive inputs. However, not restricting input length locally can still waste application resources, trigger API rate limits, or cause unexpected errors in downstream tools.
**Prevention:** Always implement both explicit HTML `maxLength` attributes on text inputs and equivalent string length checks in the component's validation logic to enforce reasonable constraints before payload construction.
